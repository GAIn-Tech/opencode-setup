## Acceptance Criteria for Net-New Skills

A proposed skill must demonstrate:

- **+2pp** one-pass correctness (absolute improvement), **OR**
- **−20%** switch rate (relative reduction)

**AND** must not breach any threshold above.

## Rollback Boundary

If a skill merge causes any threshold breach detected in CI:

1. Revert the merge commit immediately.
2. Re-run `node scripts/run-skill-routing-gates.mjs` to confirm thresholds restored.
3. Open issue with evaluator output attached for the proposer to address.

**Rollback command**:
```bash
git revert <merge-sha> --no-edit
node scripts/run-skill-routing-gates.mjs --evidence .sisyphus/evidence/skill-routing-governance/
```
